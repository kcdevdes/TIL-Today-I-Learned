# 스프링 컨테이너와 스프링 빈

## 스프링 컨테이너 생성

- ApplicationContext = 스프링 컨테이너
- ApplicationContext = Interface
- XML-based, Annotation기반 자바 설정 클래스
- new AnnotationConfigApplicationContext(AppConfig.class)는 구현체

## 스프링 컨테이너 생성 과정

1. 생성
    1. 구성 정보 지정 (AppConfig.class)
2. 스프링 빈 등록
    1. 빈 이름 - 빈 객체
    2. memberService - MemberServiceImpl@x01 → 객체 등록
    3. 빈 이름은 항상 다르게!
3. 스프링 빈 의존 관계 설정 - 준비
4. 스프링 빈 의존 관계 설정 - 완료 (싱글톤 객체)
    1. 스프링은 빈을 생성, 의존관계 주입하는 단계가 나누어짐.
    2. 자바 코드로 스프링 빈을 주입 → 의존 관계 주입 한 번에 처리됨.

## 스프링 빈 조회 - 상속관계

- 부모 타입으로 조회하면 자식 타입도 함께 조회한다.
- Object타입으로 조회하면 모든 스프링 빈을 조회한다.

## BeanFactory

- 스프링의 최상위 인터페이스
- 스프링 빈을 관리하고 조회하는 역할
- getBean()을 제공

## ApplicationContext

- BeanFactory 기능을 모두 상속 받아서 제공
- 어플리케이션 개발 시, 빈을 관리하고 조회하는 기능 + 수많은 부기가능을 필요로함
    - 메시지소스를 활용한 국제화
        - 한국에서 접속 시 한국어, 영어권에서 접속 시 영어로 출력
    - 환경변수
        - 로컬, 개발, 운영을 구분하여 처리
    - 애플리케이션 이벤트
        - 이벤트를 발행하고 구독하는 모델을 지원
    - 편리한 리소스 관리
        - 파일, 클래스 패스, 외부 등에서 리소스를 편리하게 조회

## XML 형식 코드 지원

- Java Config, XML, Groovy 지원
- AppConfig.class, appConfig.xml, [appConfig.xxx](http://appConfig.xxx) 등등등 다양한 포맷 지원
- Java: new AnnotationConfigApplicationContext(AppConfig.class)
- XML: 잘 사용 안 함, XML을 사용할 시 컴파일 없이 빈 설정 정보를 변경할 수 있는 장점이 있음.
